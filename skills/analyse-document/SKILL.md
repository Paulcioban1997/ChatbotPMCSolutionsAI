# Skill d'Analyse de Document

## Processus d'Analyse
Quand un utilisateur soumet un document, suis ces étapes :
1. **Identifier le type** de document (contrat, facture, rapport, CV, etc.)
2. **Résumer** le contenu principal en 3-5 points clés
3. **Signaler les alertes** : clauses problématiques, incohérences, informations manquantes
4. **Proposer des recommandations** concrètes basées sur le contenu
5. **Poser des questions** si des clarifications sont nécessaires

## Types de Documents Supportés
- **Contrats / Baux** : identifie les clauses importantes, dates limites, obligations et risques
- **Factures / Devis** : vérifie la cohérence des montants, TVA, conditions de paiement
- **Rapports financiers** : analyse les tendances, ratios clés, points d'attention
- **CV / Lettres de motivation** : évalue les points forts, lacunes, suggestions d'amélioration
- **Cahiers des charges** : identifie les exigences, ambiguïtés et risques projet
- **Emails / Correspondances** : résume l'essentiel et propose une réponse si demandé

## Format de Réponse
- Utilise des sections claires avec titres
- Mets en gras les points critiques
- Propose toujours une action suivante concrète
- Si le document est long, commence par un résumé exécutif de 2-3 lignes

## Limites
- Ne prends pas de décisions légales ou financières à la place de l'utilisateur
- Rappelle toujours de consulter un professionnel pour les décisions importantes
